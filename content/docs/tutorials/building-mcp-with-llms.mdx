---
title: 使用LLM构建MCP
---
使用LLM 加速 MCP 开发，例如：Claude！

本指南将帮助您使用 LLMs 构建自定义的模型上下文协议（MCP）服务器和客户端。本教程将以 Claude 为示例，但您也可以使用任何前沿 LLM。

## 准备文档 ##
开始之前，请先收集必要的文档，以帮助 Claude 理解 MCP：

1. 访问 https://modelcontextprotocol.io/llms-full.txt 并复制文档全文 
2. 导航到 MCP TypeScript SDK 或 Python SDK repository 
3. 复制 README 文件和其他相关文档 
4. 将这些文档粘贴到您与Claude的对话中 
## 描述您的服务器 ##
提供文档后，向 Claude 清楚地描述您想构建什么样的服务器。请具体说明：

- 它将公开哪些资源 
- 它将提供哪些工具
- 它是否需要提供哪些提示词
- 它需要与哪些外部系统交互 

例如：
```js
建立一个 MCP 服务器，它可以：
- 连接到我公司的 PostgreSQL 数据库
- 将模式或表等作为资源公开
- 提供运行只读 SQL 查询的工具
- 提供常见数据分析任务的提示词
```
## 使用Claude ##
当使用Claude来构建MCP 服务器时：
1. 先从核心功能开始，然后逐步迭代以添加更多功能
2. 代码中遇到任何不理解的部分，请Claude解释
3. 根据需要，请求修改或改进
4. 让Claude帮您测试服务器，并处理边界场景

Claude可以帮助实现所有关键的 MCP 功能：
- 资源管理和暴露 
- 工具定义和实现
- 提示词模板和处理程序
- 错误处理和日志记录
- 连接和传输设置

## 最佳实践 ##
使用 Claude 构建 MCP 服务器时：

- 将复杂的服务器程序分割成小块
- 在继续之前彻底测试每个组件
- 牢记安全性--验证输入并适当限制访问权限
- 代码的文档完善，以便日后维护
- 仔细遵循 MCP 协议规范

## 下一步 ##
Claude帮助您构建服务器之后：

1. 仔细查看生成的代码
2. 使用 MCP 检查器工具，测试服务器
3. 将其连接到 Claude.app 或其他 MCP 客户端
4. 根据实际使用情况和反馈进行迭代

请记住，随着需求的不断变化，Claude可以帮助您修改和完善服务器。

需要更多指导？只需向 Claude 提出具体的问题，例如有关实现 MCP 功能或实现功能过程中遇到的问题。
